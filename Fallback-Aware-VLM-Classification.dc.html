01

How the method works

A single VLM pass is overconfident: forced to pick from a fixed label set, it will name something even for out-of-set images. We run two independent reasoning paths over the same label list and only trust the answer when they converge.

VISUAL ARM

Look, then label

The VLM classifies the pixels directly against the label list — fast, but anchored on low-level appearance and prone to confident near-misses.

⇄
TEXTUAL ARM

Describe, then label

The VLM first writes a free-form summary of the image, then a classifier reasons over that text and the same labels — anchored on semantics, not pixels.

→
DECISION

Agree, or abstain

Accept as Known only if both arms name the same label and it's in the set. Any disagreement triggers fallback → Unknown.

Acceptance indicator
δ̂  =  𝟙[ yvision = ytext ]  ·  𝟙[ yvision ∈ 𝒞 ]
δ̂=1arms agree and the label is in set 𝒞 → accept as Known.
δ̂=0otherwise → fallback, mark Unknown and abstain.

03

Two ways to be wrong

A good fallback rule walks a line between accepting too much and rejecting too much. The consistency check is tuned to cut overconfidence hard while keeping over-rejection in check.

↑

Overconfidence error

An unknown image gets accepted as a known class. A single VLM pass does this constantly — it has no way to say "none of these."

tractor 🠖 truck accepted ✗
Caught by us: the textual arm reads "agricultural vehicle, large rear wheels" and says car, not truck — disagreement → Unknown.
↓

Over-rejection error

A genuinely known image gets rejected as Unknown because the two arms split on fine-grained classes that look alike.

fox: vis=fox / txt=coyote rejected ✗
The cost: abstaining safely sometimes means turning away a label we could have gotten right. Balance ratio R tracks this trade-off.
